Тестовое задание:

Необходимо создать C++ класс, который представляет собой потоки и очереди задач (раздающий задания поток, рабочие потоки).
Рабочий поток ожидает появление задачи в очереди (в виде лямбда-функции)
и выполняет лямбда-функцию, а результат выполнения передает в другую лямбда-функцию, которую помещает для выполнения в очередь
раздающего потока.

С помощью созданной конструкции в качестве примера нужно решить следующую задачу:

Имеется:
1. Буфер1 со случайным содержимым с заданным размером
2. Буфер2 для приема результата, такого же размера как Буфер1
3. Раздающий задачи поток
4. N потоков исполнения задач

Задаваемые в параметрах вызова исполняемого файла параметры задачи:
1. Размер буфера (до 100 мегабайт)
2. Количество потоков исполнения задач

В раздающий поток на вход поступает буфер1, раздающий поток должен распределить содержимое буфера1
на N потоков исполнения частями по 256 байт. При этом одновременно в очереди каждого потока исполнения должно быть не более 10 задач.
Добавление задач в очередь, раздающим потоком, должно осуществляться по событию завершения очередной задачи (за исключением первичного наполнения очереди).
Каждый потоков исполнения должен провести операцию XOR содержимого части буфера1 с числом 0xF
и поместить результат в соответствующую часть буфера2.

В раздающий поток на вход поступает буфер2, раздающий поток должен распределить содержимое буфера2
на N потоков исполнения, каждый из которых должен провести операцию XOR содержимого части буфера2 с числом 0xF
и сравнить результат с соответствующей часть буфера1.
Результат (OK - если байты совпадают и FAIL если не совпадают) вывести в консоль из раздающего задачи потока с указанием адреса части буфера для которой проведено сравнение.

Результат – программа компилируемая gcc с помощью make под Linux x86_64, принимающая параметры в командной строке и выводящая результат в консоль.

Размер буфера задается в байтах.
Буфер1 программа формирует сама заданного размера, содержимое случайные величины.

Обработка данных должна быть полностью асинхронной, ожидание завершения только в конце программы

Пример запуска программы (названия параметров любые, можно и без названий):

./app -size 1048576 -threads 4
